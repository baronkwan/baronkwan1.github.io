Enumerable methods - Enumerable#map

Established on Jan. 10, 2016

Enumerable is one of the module that make Ruby powerful. It is a whole pack of methods that prewriten for Rubyists to take and get access to. It beautifully replace the use of for loops. Provided clarity for reader. In this article, I would like to talk about one of the most useful Enumerable method - #map (alias for collect).

Lets look at the example below:


> [1,2,3].map { |i| i*i }
==> [1,4,9]


Each element in the array is modified by the block passed into the map method. When the iteration is done, it will return a new array with the result of running the block once for every element in the array. #map will run your block and give you an object filled with whatever your block returned each time. And if there is no block given after calling #map, an enumerator will be returned instead.

To get a better sense of #map/#collect, you can picture it as remapping your original array through some sort of transformation. Or doing something to each element of the array and collecting the results at the end. #map and #collect are the same thing, just called differently.
