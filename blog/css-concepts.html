    Absolute, Relative, Fixed and Static Positioning: How do they differ?


    Established on Dec. 18, 2015

    CSS Positioning is not hard but some of its concepts were abstract and mostly misused. After reading articles about CSS positioning, I would like to share the difference of the four positioning values of CSS positioning.


    Static: Every single page element is positioned in static by default. It means the element will flow into the page as it normally would.

    Relative: What it actually means is "relative to itself". If one set position: relative; but no other position attributes (top, bottom, left, right) it will show no effect on its positioning at all. It will be exactly as it would be as setting to static. One more thing should be mentioned is Relative also limits the scope of absolutely positioned child elements.

    Absolute: Similar to Relative, you want to use position attributes (top, bottom, left, right) to set the location on the page. The element that positioned as absolute will be positioned relatively to the it parent which was positioned relatively or absolutely. If no such parent was positioned before it. It will trace all the way back to the < html > element. Meaning it will be positioned relative to the page itself. Absolutely positioned elements are removed from the flow of the page.

    Fixed: A fixed position element is positioned relative to the viewport, or the browser window itself. The viewport doesn't change when the window is scrolled, so a fixed positioned element will always stay at the same position.
